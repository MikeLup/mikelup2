Reflection

Lessons Learned

We learned to always have the end user in mind when designing an interface. By focus on solving the problem you set out to solve and not overcomplicating the design with too many features we learned some of the better ways to have a better product.

What went well:

Some of the better parts of the project was the interaction witht he target user. Gettign this input allowed for use to implement the better functions for the user as well as making it a layout that the user finds easy to use.

What could be improved:

We could have put more of a focus on some of our storyboards. While we had good boards, there were times where more user feedback was needed and using some of these in storyboards could have helped more.

Impact:
